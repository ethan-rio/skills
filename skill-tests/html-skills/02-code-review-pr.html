#4127 — Replace token-bucket limiter with sliding-window counter

1 critical 2 warnings 1 nit · branch erin/sliding-window · base main

What this PR does

Swaps the per-route token-bucket rate limiter for a Redis-backed sliding-window counter. Goal: stop the 1-second burst-then-pause pattern that's been tripping our SLO alerts. Affects every authenticated API route via middleware.

Focus areas: concurrency in Acquire and the new Redis Lua script. Skim the test additions.
